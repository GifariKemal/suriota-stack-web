1. **[HIGH · S effort · +5-8 pts each]** Inject **Service JSON-LD** on every pillar (see template in §7). Each pillar gains the schema-axis differential (15 vs 8 = +7 pts).
2. **[HIGH · S effort · +1-2 pts each]** Tighten the **title tag** to 50-65 chars on 5554 and 5558 (currently 24 chars). Suggested: `Industrial IoT & System Integration — SURIOTA Indonesia` and `SURGE SaaS Industrial Monitoring Platform — SURIOTA`.

4. **[HIGH · S effort · +2 pts]** Add **breadcrumb JSON-LD** (Home › Services › <pillar>) — improves rich-snippet eligibility.
5. **[MEDIUM · S effort · +5 pts each on internal-link signal]** Add **Related Projects** component (§6 spec) — 4-6 cards per pillar, all sourced from the 64-item portfolio. This also raises cross-pillar via projects linking back.
6. **[MEDIUM · S effort]** Add `<meta name="robots" content="max-snippet:-1, max-image-preview:large, max-video-preview:-1">` if not already sitewide.
7. **[LOW · S effort]** Confirm OG image is at least 1200×630 PNG/JPG (not a placeholder) for the pillar share-card.

### Strategic Improvements (larger effort, higher ceiling)

1. **[HIGH · L effort · +20 pts each — single biggest lever]** **Triple body content per pillar** to 1,500-2,500 words. Concrete H2 outlines provided in §3 per pillar. This single axis is what holds every pillar at 5/25 instead of 25/25.
2. **[HIGH · M effort · +5-10 pts cumulative]** Build an internal-link mesh: each pillar links to (a) the other 4 pillars (Why pillar X vs Y micro-section), (b) 4-6 specific portfolio posts, (c) the contact form with deep-linked anchor.
3. **[MEDIUM · M effort]** Rewrite portfolio post titles for SEO clarity — currently many are bilingual hybrids (e.g. "jasa-maintenance-webmail-webmail-maintenance-pt-hijrah-travel"). Cleaner slugs + H1 would help pillar→portfolio link relevance.
4. **[MEDIUM · M effort]** Add **FAQ JSON-LD** on each pillar covering 5-7 buyer questions (e.g. "How long does an IoT pilot take?", "Which protocols do you support?"). Adds 5-8 H3 and triggers FAQ rich result.
5. **[MEDIUM · M effort]** Categorize 64 portfolio items into a dedicated `case_study` CPT (or at minimum a `pillar-link` ACF taxonomy) — enables auto-rendered related-projects without hardcoding.
6. **[LOW · L effort]** Image optimization audit — confirm all portfolio thumbnails are WebP and lazy-loaded; pillar hero images should use `fetchpriority="high"`.

## 6. Suggested Related Projects Component Spec

**Class namespace:** `.sxp-related` (sxp = SURIOTA Pillar)  
**Placement:** After the last service H2 / above the contact CTA on every pillar.  
**Eyebrow micro-copy (i18n):** `RELATED WORK` (EN) · `PROYEK TERKAIT` (ID) · `相关案例` (ZH)  
**H2 micro-copy:** `See how we have delivered this` (EN) · `Lihat hasil kerja kami` (ID) · `查看我们的成果` (ZH)

```html
<section class="sxp-related" aria-labelledby="sxp-related-heading">

```

### Design rules
- Card count: 4 on mobile (stacked), 4 (2×2) on tablet, 6 (3×2) on desktop.
- Use `clamp(...)` font sizing consistent with `sx-` design system tokens.
- Image: 16:9 cover, lazy-loaded, WebP. Min 480×270, max 1280×720.
- Pill colour token: `var(--sx-accent)` for matching pillar.
- Focus state visible (≥2px outline). Cards must be reachable via Tab.
- Each card is one `<a>` wrapper to keep semantics simple; do not nest links.

## Appendix A — Methodology

- **Portfolio inventory:** Fetched via `GET /wp-json/wp/v2/posts?per_page=50&page={1..3}&_embed`. The portfolio page renders client-side from this endpoint (see JS in `/portfolio/` source).
- **Pillar audits:** Headless Chromium (Playwright) navigated to each EN pillar URL with `networkidle` wait, then extracted: title, meta description, OG/Twitter meta counts, canonical, hreflang, viewport, JSON-LD scripts (parsed for @type), visible word count in `main`/`article` scope, H1/H2/H3 counts scoped to the same container, total `<a>` and image elements with `alt` coverage.
- **Pillar mapping:** Rule-based on title + excerpt + WP category. Each item can map to multiple pillars (primary = first match).
- **Scoring:** 10-axis weighted rubric defined in §3, total 100. Scoring code at `audit/pillar-v2-snapshots/_tmp/pillar_audit.json`.
- **NOT audited in this run (out of scope):** ID/ZH pillar translations, Core Web Vitals (LCP/CLS/INP), Lighthouse performance, axe a11y full sweep.
